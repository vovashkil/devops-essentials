Deploy application using AWS Developer Tools

Introduction

In this lesson, you will see me deploying a Java application using several different AWS services.

Manually Deploy using AWS Developer Tools

Follow these steps to deploying a Java application on an EC2 instance -

  • IAM - Create a new IAM User and give it access AWSCodeCommitFullAccess and all AWS resources. Login with this new user.
  • Prerequisites - Ensure Prerequisites have been established
  • EC2 - Create an EC2 Key Pair
  • S3 - Create an S3 bucket to store (version it)
  • IAM - Create an IAM role for CodeDeploy using the AWSCodeDeployRole Policy
  • CloudFormation - Launch a CloudFormation stack. that launches an EC2 instance and installs the CodeDeploy agent on it. Use https://s3.amazonaws.com/www.devopsessentialsaws.com/samples/training/CodeDeploy_SampleCF_Template.json when launching the stack for the Specify an Amazon S3 template URL option
  • CodeBuild - Create a CodeBuild project using this as the source: https://github.com/stelligent/aws-codedeploy-sample-tomcat and select Zip in the Artifacts packaging section of Advanced Settings
  • CodeDeploy - Manually create a CodeDeploy application and Deployment Group
  • CodeDeploy - Deploy a new version and use the S3 bucket where the zip from the CodeBuild was stored
  • CodeDeploy/EC2 - Copy the public DNS and pasted into browser
  • SNS - Manually create an SNS Topic and Email Subscription
  • IAM/CodeCommit - Assign Git credentials for CodeCommit in IAM
  • CodeCommit - Manually create a CodeCommit repository
  • CodePipeline - Manually create a CodePipeline pipeline
