Course Introduction

Introduction

DevOps Essentials on AWS is a solutions-focused LiveLesson. It's the right video course if you're some type of software or DevOps-focused engineer or architect interested in learning how to use AWS Developer Tools to create a full-lifecycle softwre delivery solution. This LiveLesson shows you how to implement deployment pipelines using the AWS Developer Tools. The focus of the LiveLesson is on deployment pipeline architectures versus software architectures (e.g., monoliths, microservices, FaaS, BaaS, etc.). This LiveLesson is for you if you want to see various sample solutions for end-to-end automation of your infrastructure and pipelines.

The DevOps Essentials LiveLessons has a heavy focus on the use of AWS Developer Tools and other services, including:

  • AWS CloudFormation - a tool you can use to automate the provisioning of AWS resources
  • AWS CodePipeline - which provides the capability to orchestrate your software delivery workflows
  • AWS CodeCommit - a Git-based version-control repository
  • AWS CodeBuild - a fully-managed service for running automated builds, tests, and static analysis
  • AWS CodeDeploy - a fully-managed service for deploying software to environments

AWS CodePipeline is the primary tool used in all the examples of this LiveLesson. All of the solutions are automated via CloudFormation. There are seven examples that use many of the AWS Developer Tools. All of the examples are available from a GitHub repository that will be referenced throughout this LiveLesson. All of the examples are based on the static website that you're viewing now and will be used to walk viewers through the lesson. The seven example solutions are:

  • Static Website (i.e., AWS S3 hosted)
  • EC2 (traditional EC2 deployment with CodeBuild and CodeDeploy on EC2 instances)
  • OpsWorks
  • Elastic Beanstalk
  • ECS
  • Serverless
  • CodeStar

For each of the lessons, this LiveLesson will add additional complexity by adding additional stages, actions, databases, and services into the static website solution. While the LiveLesson predominantly uses AWS tools, there will be some third-party integratations and open-source tools incorporated into the examples. You can fork the GitHub repository to make your own changes.

In Lesson 1, you will be able to describe the basic components and tools of a deployment pipeline, describe DevOps, describe patterns of effective culture, learn team organization patterns, and the basics of establishing Continuous Delivery to deliver software on demand.

In Lesson 2, you will configure the course prerequisites on your computer, manually provision the static website, describe how to orchestrate software workflows in CodePipeline, store software assets in CodeCommit, Build and test software in CodeBuild, describe the architecture and launch the solution, provision AWS resources in CloudFormation, and orchestrate solution in CodePipeline.

In Lesson 3, you will describe basic EC2 features, describe how to deploy applications using CodeDeploy, describe the architecture and launch the solution, provision AWS resources in CloudFormation, and orchestrate the EC2-based solution in CodePipeline.

In Lesson 4, you will describe the basic features of OpsWorks, describe the architecture and launch the solution, provision AWS resources in CloudFormation, and orchestrate the solution in CodePipeline

In Lesson 5, you will describe basic features of Elastic Beanstalk, describe the architecture and launch the solution, provision AWS resources in CloudFormation, orchestrate solution in CodePipeline

In Lesson 6, you will describe basic features of EC2 Container Service (ECS), Describe the architecture and launch the solution, provision AWS resources in CloudFormation, orchestrate solution in CodePipeline

In Lesson 7, you will describe basic features of Serverless, describe the Serverless Application Model, describe the architecture and launch the solution, provision AWS resources in CloudFormation, and orchestrate solution in CodePipeline

In Lesson 8, you will launch solution from Project Template, configure Dashboard for Project, manage code changes, deploy software, orchestrate software using Pipeline, manage team members, and extend solution using Extensions.

There will be a lesson for each of seven examples listed above. Each lesson will follow a similar format:

  • Introduction - Introduce the learning objectives
  • Prerequisites - Describe any setup/configuration the viewer needs to perform
  • Architecture - Show the architecture and capability to click a Launch Stack button. To see an example, go to Lesson 2.2 (Static Website: Architecture)
  • Code - Walk through the CloudFormation template(s) in detail since this is the basis for the automation of the entire solution. Will also view buildspec.yml, build, test and deploy scripts, appspec.yml, etc.
  • Stages - For most lessons, there will be six stages in a pipeline - Source, Build, Dev, PreProd, Canary, and Production. There will be a separate sublesson for each of the stages that describe the various actions, transitions, etc. that make up the pipeline. I describe each of the stages next.
  • Monitoring - Using CloudWatch to view pipeline and production activity
  • Summary - A summary of what you learned
  • Quiz - You can test your knowledge with a quiz at the end of the lesson

The stages are a focal point of each lesson. As I mentioned previously, most examples will include a six-stage pipeline as described below:

  • Source - Will use GitHib and CodeCommit as source providers
  • Build - Unit tests, style checkers, etc.
  • Dev - Deploy and integration tests - used to test the running service.
  • PreProd - Deploy and integration tests - used to test the combination of new software and production config. Looks like prod, uses prod config, but doesn't receive customer traffic.
  • Canary - OneBox Deployment (deploy and monitor)
  • Production - Deploy to production (zonal, regional, etc.)

Other tools and features to cover include:

  • Lambda
  • CodePipeline gates, transitions, and approvals
  • Parameter Store
  • Route 53
  • ELB
  • Blue/Green Deployment in CodeDeploy
  • VPC
  • Auto Scaling
  • IAM

Specific Lessons:
