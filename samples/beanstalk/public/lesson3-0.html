Tools

  • CodePipeline - Defines the stages, actions, gates and transitions in the software delivery workflow

  • CodeBuild - Run build, unit tests, and static analysis for Java application
  • CodeDeploy - Use Blue/Green Deployment with CodeDeploy

Stages

  • Source Stage - Version-Control Repository. Native Support for S3, CodeCommit, and GitHub. Will be using GitHub and CodeCommit in this course
  • Build Stage - with unit tests, style checkers, etc.
  • Dev Stage - deploy and integration tests) - used to test the running service.
  • PreProd Stage - PreProd (deploy and integration tests) - used to test the combination of new software and production config. Looks like prod, uses prod config, but doesn't receive customer traffic.
  • Canary Stage - (OneBox) Deployment (deploy and monitor).
  • Production Stage - deploy.
