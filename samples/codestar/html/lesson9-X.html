9.X Course Notes

Vision

This course will be solution focused. I'll spend a little bit of time at the beginning describing DevOps, CI/CD, and AWS. In re:Invent terms, this is close to a 300-level session. Not super deep, but also makes the assumption that they are engineers and architects, are aware of the cloud, and want to learn how to do continuous delivery, etc. on the AWS platform in a fully-automated manner.

I'll focus purely on deployment pipeline architectures and implementation and not on software architectures. Will let the viewer determine how they can use it with various software architecture types (e.g. monoliths, microservices, FaaS, BaaS, etc.). I might make brief references to software architectures but I think anything other than that won't provide the proper level of depth and lessen the course's focus.

With very few exceptions, I plan to use only AWS tools. These tools include:

  • CloudFormation
  • CodePipeline
  • CodeCommit
  • CodeBuild
  • CodeDeploy

CodePipeline will be the primary tool for all the examples. All of the solutons will be automated via CloudFormation. There will be seven examples that use many of the tools above. All of the examples will be available from our GitHub repository. All seven examples are based on the static website. This website is the one you're viewing now and will be used to walk viewers through the lesson. The seven example solutions are:

  • Static Website (i.e. S3 hosted)
  • EC2 (traditional EC2 deployment with CodeBuild and CodeDeploy on EC2 instances)
  • OpsWorks
  • Elastic Beanstalk
  • ECS
  • Serverless
  • CodeStar

I'll add additional complexity to the "static website" to include Java builds, services running on containers, databases, etc. but all the examples will use this website to make the live changes. I plan to incorporate the use of some third-party integratations, open-source tools into the actions. Viewers can fork the repository to make their own changes.

There will be a lesson for each of seven examples listed above. Each lesson will follow a similar format:

  • Introduction - Introduce the learning objectives
  • Prerequisites - Describe any setup/configuration the viewer needs to perform
  • Architecture - Show the architecture and ability to click a Launch Stack button. To see an example, go to Lesson 2.2 (Static Website: Architecture)
  • Code - Walk through the CloudFormation template(s) in detail since this is the basis for the automation of the entire solution. Will also view buildspec.yml, build, test and deploy scripts, appspec.yml, etc.
  • Stages - For most lessons, there will be six stages in a pipeline - Source, Build, Dev, PreProd, Canary, and Production. There will be a separate sublesson for each of the stages that describe the various actions, transitions, etc. that make up the pipeline. See each stage described below.
  • Monitoring - Using CloudWatch to view pipeline and production activity
  • Summary - A summary of what they learned
  • Quiz - Viewers can test thier knowledge with a 5-10 question quiz of the lesson

The stages will be a focal point of each lesson. As mentioned above, most examples will include a six-stage pipeline as described below:

  • Source - Will use GitHib and CodeCommit as source providers
  • Build - Unit tests, style checkers, etc.
  • Dev - Deploy and integration tests - used to test the running service.
  • PreProd - Deploy and integration tests - used to test the combination of new software and production config. Looks like prod, uses prod config, but doesn't receive customer traffic.
  • Canary - OneBox Deployment (deploy and monitor)
  • Production - Deploy to production (zonal, regional, etc.)
