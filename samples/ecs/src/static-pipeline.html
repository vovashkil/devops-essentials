Static Website: Deployment Pipeline

Narrative

In this sublesson, you will walk dive into the Source Stage for this pipeline. Before we do that, though, let's have a look at AWS CodePipeline in a bit more detail. In CodePipeline, Stages are composed of a collection of Actions. Actions actually perform the work in a pipeline. Within Actions, you'll see integration with other tools, running commands to build, test, deploy, and release your service. Currently, there are six Action types: Source, Build, Test, Deploy, Invoke, and Approval. In this sample, we're just using two types: Source and Build. You can run actions in parallel. A stage is not complete until all actions are successful.

You can enable and disable transitions to acts as gates between stages and several other features but let's not get ahead of ourselbes as you'll be learning more about this and other features of CodePipeline in other lessons.
